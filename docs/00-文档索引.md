## 三层关系
```
小程序(tennisyy)  ──BLE GATT写──►  BLE模块  ──UART2透传Modbus RTU──►  STM32固件
        ▲                                                                          │
        │ 同一套 Modbus RTU 协议（CRC16/寄存器地址/球参数编码完全一致）            │
        │                                                                          ▼
Windows程序(TennisTraining) ──串口/BLE──►  STM32固件（ModBus_unpack → BL_Read_DATA → 电机驱动）
```
- **协议一致性**：Windows 端的 `ModbusCrc16`/`BallParameterEncoder`/`LauncherCommands` 与小程序 `util.CRCMB16`/`processBalls`/`serveControl.vue` 命令、以及固件 `Bluetooth.h BL_Read` 字段三方对齐，已用标准向量 "123456789"→0x4B37 与启动帧 `010600030001` 验证。
- **球参数编码**：左右旋 -15..15→30..60；旋转 上旋{4,3,2}/平击5/下旋{6,7,8}（固件 `Ball_SPIN` 支持 0~4 上旋/5 平/6~10 下旋，App 为其子集）。
- **故障码**：三方一致（0001~0008 电机堵转/故障/堵球）。

## 快速验证
```bash
cd TennisTraining
dotnet build TennisTraining.sln
dotnet run --project src/TennisTraining.ModuleDemos -- all   # 5模块自检+CRC 全部通过
dotnet run --project src/TennisTraining.App                   # 启动 Windows 主程序
```
