# 小程序接口说明（tennisyy / uni-app）

> 资料来源：`3 软件/tennisyy`（Vue2 + uni-app，已测试版本）。本说明基于其真实源码梳理，供 Windows 端与其它客户端对接参考。

## 1. 工程概览
- 框架：uni-app（Vue2），可编译微信小程序 / iOS / Android。
- 入口：`App.vue`（全局数据 globalData + 消息总线 eventBus）、`main.js`、`pages.json`。
- 通信方式：BLE 蓝牙（GATT）透传 Modbus RTU 帧与下位机交互。
- 业务核心页：`pages/serveControl/serveControl.vue`（发球控制）、`pages/lessonSetting/lessonSetting.vue`（课程设置）、`pages_two/addLesson` 与 `lessonChange`（自定义课程）。

## 3. BLE GATT 接口
定义于 `App.vue` globalData：
| 项 | 值 |
|---|---|
| Service UUID | `55535343-FE7D-4AE5-8FA9-9FAFD205E455` |
| 读/Notify 特征 UUID | `49535343-1E4D-4BD9-BA61-23C647249616` |
| 写特征 UUID | `49535343-8841-43F4-A8D4-ECBE34729BB3` |
| MTU（Android） | 247 |
连接流程（`utils/bleUtil.js`）：`createBLEConnection` → `getBLEDeviceServices`（找上述 Service）→ `getBLEDeviceCharacteristics`（绑定读写/Notify 特征）→ `notifyBLECharacteristicValueChange` 订阅上行 → `writeBLECharacteristicValue` 下发。
- 数据以 **hex 字符串**经 `string2Buffer` 转 ArrayBuffer 写入；上行 Notify 在 `onBLECharacteristicValueChange` 回调中以 ArrayBuffer 接收，`buf2hex` 转字符串后 CRC 校验（`util.checkCrc`）。

## 4. 通讯协议（Modbus RTU over BLE）
帧格式：`[地址 01][功能码][起始地址 H L][数量/值][数据...][CRC L][CRC H]`，CRC 为标准 Modbus CRC16（`util.CRCMB16`，低字节在前）。所有命令均由 serveControl.vue 构造后 `bleWrite` 发送。

### 4.1 设备控制（功能码 0x06，寄存器 0x0003）
`changeDeviceState(flag)`：`01060003` + 值 + CRC
| flag | 值 | 含义 |
|---|---|---|
| 0 | 0000 | 停止 |
| 1 | 0001 | 开始 |
| 2 | 0002 | 故障 |
| 3 | 0003 | 暂停 |

### 4.2 微调（功能码 0x06，寄存器 0x001D）
`0106001D` + 值 + CRC
| 值 | 含义 | | 值 | 含义 |
|---|---|---|---|---|
| 0001 | 弧度+ | | 0004 | 水平- |
| 0002 | 弧度- | | 0005 | 球频+ |
| 0003 | 水平+ | | 0006 | 球频- |

### 4.3 会话配置（功能码 0x10，起始 0x001A，5 寄存器）
`goModel()`：`0110001A 0005 0A` + 数据 + CRC
数据 5 寄存器 = `[发球总时间, 课程循环数, 组循环数, 微调(0000), 组循环间隔]`，按模式（1组/2计时/3自由）填充：
- 组模式：总时间=0000(不启用)，组循环数=实际值
- 计时/自由模式：总时间=计时秒数或0000，组循环数=FFFF(不限制)

### 4.4 球参数下发（功能码 0x10，起始 0x0021）
`sendAllBalls()`：`util.buildSendList(finalBalls)` 生成 3 包（每包 15 球，共 45 球，不足补空球）。
每球 6 字节 = `[球速, 弧度, 左右旋(已映射), 球频, 旋转(已映射), 00]`。
- 球参数预处理 `util.processBalls`：左右旋 -15..15 → +45（30..60）；旋转 上旋`4-floor((s-1)/2)`∈{4,3,2}、平击=5、下旋`6+floor((s-1)/2)`∈{6,7,8}。
- 发球顺序：`flag_order` 1顺序 / 2随机（`util.shuffleArray`）。
- 发完最后一包后调用 `changeDeviceState(1)` 启动。

### 4.5 固件升级（OTA）
`upgrade.vue`：将 bin 转 base64 分片，通过自定义帧（`sendPackageInfoCode`/`sendPackageSliceCode` + CRC）下发，含版本查询 `queryVersionCode`、进入升级模式 `startUpgradeModeCode`。

## 5. 课程数据模型（data/courseData*.js）
每个摆放位(1-6)对应一个 courseDataN.js，结构：
```
{ version, lessonData:[...], lessonDataTwo:[...], lessonDataMore:[...] }
```
课程对象字段：
| 字段 | 含义 |
|---|---|
| name | 课程名称 |
| devPosition | 1底线左/2中/3右/4T字线/5左抛球位/6右抛球位 |
| isCollect | 是否收藏 |
| currentDiff | 当前难度 1/2/3 |
| diffOne/diffTwo/diffThree | 三档难度，每档为球参数数组 |
球参数：`spinType`(1上旋/2平击/3下旋)、`leftRight`(左右旋 -15..15)、`speed`、`rate`(球频)、`height`(弧度)、`hand`(1左/2右)、`spin`(上下旋强度)。
工具类 `utils/lesssonDataUtils.js`：`initLessonToStorage`（首次写入 storage）、`getLessonDataById`、`getCollectedData`、`checkUpdateLessons`/`updateLessonData`（按版本号升级，保留收藏状态）。

## 6. 本地存储键（globalData）
`save_ble_name`、`save_user_lesson`(自定义课程)、`save_diff`、`save_favorite`、`save_lesson_one..six`(6 摆放位内置课程)、`save_loop_group_count`、`save_loop_group_time`、`save_serve_mode`(1组/2计时/3自由)、`save_serving_order`(1顺序/2随机)、`save_mode_group_num`、`save_mode_time_num`、`save_course_version`、`save_quick_type/id`。

| 0002 | 下发球电机堵转 | | 0006 | 旋转电机故障 |

## 8. 消息总线与状态
`App.vue` 内置 `eventBus`（on/emit/off），用于跨页通信（如 `onBledata` 升级数据回传）。`bleConnectionStatus`（disconnected/connecting/connected）驱动 UI 连接态。蓝牙断开后视为设备断电，重新开始需重发全部课程数据（`flag_ble_disconnect`/`flag_device_state` 逻辑）。
