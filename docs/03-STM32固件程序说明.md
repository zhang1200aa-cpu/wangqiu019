# STM32 固件程序说明（HJSE-1001 网球机电机驱动程序 V3）

> 资料来源：`2 电气/2.3 程序代码/网球机_CODE_V5.0_20250627/网球机电机驱动程序 - 第二版V3`。MCU：STM32F103（Cortex-M3，72MHz），Keil MDK 工程，标准外设库 STM32F10x_FWLib。固件版本 `VERSION 0x020a`（v2.10）。

## 1. 工程目录
```
网球机电机驱动程序 - 第二版V3/
├── CAR/              应用层（main.c + Keil 工程文件 CAR.uvproj）
├── CORE/             Cortex-M3 内核
├── HARDWARE/LCD/     外设：12864 LCD、按键 key、WS2812 LED、nRF24L01、bsp、Mod_APP.C(寄存器存储)、modbus.h
├── STM32F10x_FWLib/  标准外设库（GPIO/USART/TIM/ADC/DMA/SPI/Flash…）
└── SYSTEM/           系统层
    ├── delay/        delay.c 延时
    ├── motor/        motor.C(118KB，电机控制核心) + motor.h(GPIO与变量定义)
    ├── sys/          sys.c
    └── UART/         Bluetooth.c/h(BLE)、rs232.C/h(Modbus/485)、UART1/2/3、UART_B、usart1/2/3、system_dat.h
```

## 2. 主程序流程（main.c）
初始化：`SystemInit` → `delay_init(72)` → `GPIO_Initial` → 电机断电 → `Usart1_Init(115200)`、`Usart2_Init(115200)`（BLE 串口）、`Usart3_Init(9600)` → `TIM1-4_Initial` → `ADC_Configuration` → `WS2812_DMA_Init/2/3`（LED 灯带）→ `flash_read(MODBUS_Address)` 读取标定参数（RL/UD 角度校准值 `RL_Angle_ADJ`/`UD_Angle_ADJ`）→ `RL_RST_Flag`/`UD_RST_Flag` 复位（旋转/俯仰回零：`Motor5_RST`/`Motor6_RST`）。
主循环（10ms 一轮）：`Start_Run` → `Get_Adc` → `Motor1/2/3_UP`（上发球/下发球/拨球电机更新）→ `Motor5/6_UP/DOWN`（旋转/俯仰）→ `UD_Adjust`/`RL_Adjust`（角度闭环）→ `Get_Blue`/`Set_Blue`（读/写 BLE 数据）→ `MODE_Select`/`Data_SET`（模式与参数应用）→ `Motor1/2/6_BACK`（反转/退球）→ 目标计数判断（达到 `BLUET_Target_NUM` 自动停机置 STATE=0）→ 故障处理（`Error_data`→`ERROR_WRITE`，蜂鸣）→ 退球延时处理 → `delay_ms(10)`。

## 3. 电机系统（motor.h）
6 路电机，GPIO 宏定义：
| 电机 | 使能/方向引脚 | 作用 |
|---|---|---|
| M1 | PC9/PC8 (EN1/EN2) | 上发球轮 |
| M2 | PC7/PC6 | 下发球轮 |
| M3 | PC10(右)/PA15(左) | 拨球电机 |
| M5 | PD2(右)/PB3(左) | 旋转电机（左右） |
| M6 | PC12(右)/PC11(左) | 俯仰电机（上下） |
其它 IO：`Tennis_IN_State`(PC13 球检测)、`POS_State`(PC14)、限位 `POS_LIMIT1/2`(PA8/PA11)、按键 `KEY_IN_1..4`(PB9/PB6/PB5/PB4)、蜂鸣 `BEEP`(PB13)、BLE 复位 `BlueTooth_RST`(PA0)、LED1/LED2(PB15/PB14 + 数据 PB7/PB8)。
`Ball_Data` 结构：`{position, speed, RL_ANGLE, UD_ANGLE}`。
关键变量：`Ball_Speed_SET`/`UD_Angle_SET`/`RL_Angle_SET`/`Ball_FREQ_SET`/`Ball_SPIN_SET`/`MODE_SET`/`BLUET_Target_NUM`/`BLUET_Target_Count`；刷新标志 `Ball_Start_Refresh`/`Ball_Mode_Refresh`/`Ball_Data_Refresh`/`Ball_Speed_Refresh`/`Ball_RL_Refresh`/`Ball_UD_Refresh`/`Ball_SPIN_Refresh`/`Ball_BACK_Refresh`；电机电压电流 `Motor1-6_V/A`、电池 `POWER_V`、角度传感器 `Angle_RL/UD`；错误 `ERROR_Flag`、堵转检测。

## 4. BLE 数据接口（Bluetooth.h —— 核心接口）
BLE 模块经 UART2 串口接入，`Bluetooth.c` 用 AT 命令初始化（`AT`/`ATE0`/`AT+BLEMAC?`/`AT+BLENAME=HJ-<id>`），广播名 `HJ-<设备ID>`。数据模型为 `struct BL_Read`（`#pragma pack(2)`）：
| 字段 | 类型 | 含义 |
|---|---|---|
| ID_H / ID_L | u16×2 | 设备 ID |
| BATTERY | u16 | 电量 0~100% |
| STATE | u16 | 0关机 / 1运行 / 2故障 |
| NUM | u16 | 累计发球数 |
| Target_NUM | u16 | 目标发球数，达到后停机 |
| MODE | u16 | 发球模式（按地址） |
| Ball_Speed | u16 | 球速 km/h |
| Ball_ANGLE1 | u16 | 上下抛球角度 0~90° |
| Ball_ANGLE2 | u16 | 水平抛球角度 0~90°（对应 -45°~+45°） |
| Ball_FREQ | u16 | 球频 1~10 球/3s |
| Ball_SPIN | u16 | 旋转：0~4 上旋 / 5 平击 / 6~10 下旋 |
| Ball_CYCLE[14] | u16×14 | 循环参数（按地址） |
| Ball_Back | u16 | 反转参数 |
| UP_Speed / DOWN_Speed | u16 | 上/下发球轮百分比 |
| Mag_ANGLE1 / Mag_ANGLE2 | u16 | 上下/水平磁铁运动角度 |
`Get_Blue`/`Set_Blue`（motor.C）负责把 BLE 收到的帧解析写入 `BL_Read_DATA` 并触发对应 `*_Refresh` 标志，主循环据此驱动电机。该结构即为 App↔固件的寄存器映射，与小程序 Modbus 寄存器一一对应。

## 5. 通讯协议层
- `modbus.h`：`_MOD_FRAMES{addr,function,reg_addr,len,dat[250]}`，提供 `ModBus_unpack`（解析主机请求）、`ModBus_HostPack`（组响应）。即 Modbus 从站解包器。
- `rs232.h`：`Screen_PRTC{Address,Fun_Code,adds_H/L,Len_H/L}`（读寄存器协议结构）、`RS485_PRTC`（485 电机驱动器查询：电压/电流/功率/ERROR/校验和）、`Angle_PRTC`（角度协议）。函数 `Screen_Unpack`/`Screen_Pack`/`RS485_Pack`/`Screen_Pack_ALL`。`RS485_ADD`/`baudSET`/`Driver_Err` 支持多驱动器地址与波特率配置。
- `Mod_APP.C`：`Write_REG`/`Read_REG` 按 `reg_addr&0xff00` 分段处理（0x0100 设定、0x0200 时间/黑名单、0x0300 设备ID…），`FLASH_PageWriteE` 页写、`Read_File` 日志读取。寄存器 RAM 镜像 `reg_in_ram[512]`，标定参数存 Flash `0x0800F000`。

## 6. 系统数据类型（system_dat.h）
位/字节联合体：`TYPE_BIT8`/`TYPE_BIT16`/`TYPE_BIT_BYTE`/`TYPE_BYTES_WORD`；`TYPE_Motor`（12 路电机推/拉位）、`TYPE_BALL_DATA`（SPEED1/2、RL_ANGLE、UD_ANGLE）、`_SYS_REG`（继电器/数字量/输入寄存器/设备使能/本机地址）。`VERSION 0x020a`。

## 7. 数据流总结
小程序 →(BLE GATT 写)→ BLE 模块 →(UART2 串口透传 Modbus RTU)→ STM32 `ModBus_unpack` → 写入 `BL_Read_DATA` 对应寄存器 → 置 `*_Refresh` 标志 → 主循环 `MODE_Select`/`Data_SET` 应用到 `Ball_*_SET` → `Motor*_UP`/`UD_Adjust`/`RL_Adjust` 驱动电机执行 → 传感器 ADC 反馈闭环 → 故障置 `Error_data` → `Set_Blue` 上报状态。Windows 端与小程序共用同一 Modbus RTU 协议，可经串口或 BLE 直接对接该固件。

## 8. 烧录与调试
工程文件 `CAR/CAR.uvproj`（Keil），输出 `CAR/CAR.hex`（181KB）。可用 J-Link 烧录调试（`CAR/JLinkLog.txt` 为调试日志）。升级固件也可经小程序 OTA（`upgrade.vue` 分片下发 base64，固件侧 `Read_File`/写 Flash）。
