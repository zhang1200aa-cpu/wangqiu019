# 05 STM32 固件完整说明（HJSE-1001 电机驱动 V3）

> 源码：`2 电气/2.3 程序代码/网球机_CODE_V5.0_20250627/网球机电机驱动程序 - 第二版V3`。MCU STM32F103(Cortex-M3,72MHz)，Keil MDK，标准外设库。固件版本 `VERSION 0x020a`(v2.10)。

## 5.1 工程目录
```
CAR/              应用层(main.c + CAR.uvproj + CAR.hex)
CORE/             Cortex-M3 内核
HARDWARE/LCD/     12864 LCD、key、WS2812 LED、nRF24L01、bsp、Mod_APP.C(寄存器存储)、modbus.h
STM32F10x_FWLib/  标准外设库(GPIO/USART/TIM/ADC/DMA/SPI/Flash)
SYSTEM/           delay、motor(motor.C 118KB + motor.h)、sys、UART(Bluetooth/rs232/UART1-3/UART_B/usart1-3 + system_dat.h)
```

## 5.2 主程序（main.c）
初始化：`SystemInit`→`delay_init(72)`→`GPIO_Initial`→电机断电→`Usart1_Init(115200)`/`Usart2_Init(115200)`(BLE)/`Usart3_Init(9600)`→`TIM1-4_Initial`→`ADC_Configuration`→`WS2812_DMA_Init/2/3`(LED)→`flash_read(MODBUS_Address)` 读标定(`RL_Angle_ADJ`/`UD_Angle_ADJ`)→`RL_RST_Flag`/`UD_RST_Flag` 回零(`Motor5_RST`/`Motor6_RST`)。
主循环(10ms)：`Start_Run`→`Get_Adc`→`Motor1/2/3_UP`(上发球/下发球/拨球)→`Motor5/6_UP/DOWN`(旋转/俯仰)→`UD_Adjust`/`RL_Adjust`(角度闭环)→`Get_Blue`/`Set_Blue`(BLE 读写)→`MODE_Select`/`Data_SET`(应用参数)→`Motor1/2/6_BACK`(反转/退球)→目标计数判断(达 `BLUET_Target_NUM` 停机 STATE=0)→故障处理(`Error_data`→`ERROR_WRITE`+蜂鸣)→退球延时→`delay_ms(10)`。

## 5.3 电机与 GPIO（motor.h）
| 电机 | 引脚 | 作用 |
|---|---|---|
| M1 | PC9(EN1)/PC8(EN2) | 上发球轮 |
| M2 | PC7(EN1)/PC6(EN2) | 下发球轮 |
| M3 | PC10(右)/PA15(左) | 拨球电机 |
| M5 | PD2(右)/PB3(左) | 旋转电机(左右) |
| M6 | PC12(右)/PC11(左) | 俯仰电机(上下) |
其它 IO：`Tennis_IN_State`(PC13 球检测)、`POS_State`(PC14)、限位 `POS_LIMIT1/2`(PA8/PA11)、按键 `KEY_IN_1..4`(PB9/PB6/PB5/PB4)、蜂鸣 `BEEP`(PB13)、BLE 复位 `BlueTooth_RST`(PA0)、LED1(PB15+数据PB7)、LED2(PB14+数据PB8)。
`Ball_Data{position,speed,RL_ANGLE,UD_ANGLE}`。关键变量：`Ball_Speed_SET/UD_Angle_SET/RL_Angle_SET/Ball_FREQ_SET/Ball_SPIN_SET/MODE_SET/BLUET_Target_NUM/BLUET_Target_Count`；刷新标志 `Ball_Start/Mode/Data/Speed/RL/UD/SPIN/BACK_Refresh`；电机电压电流 `Motor1-6_V/A`、电池 `POWER_V`、角度传感器 `Angle_RL/UD`；错误 `ERROR_Flag`、堵转检测。

## 5.4 BLE 数据接口（Bluetooth.h — 核心）
BLE 模块经 UART2 接入，`Bluetooth.c` 用 AT 命令初始化(`AT`/`ATE0`/`AT+BLEMAC?`/`AT+BLENAME=HJ-<id>`)，广播名 `HJ-<设备ID>`。`struct BL_Read`(`#pragma pack(2)`)字段：
| 字段 | 类型 | 含义 |
|---|---|---|
| ID_H/ID_L | u16×2 | 设备 ID |
| BATTERY | u16 | 电量 0~100% |
| STATE | u16 | 0关机/1运行/2故障 |
| NUM | u16 | 累计发球数 |
| Target_NUM | u16 | 目标数，达到停机 |
| MODE | u16 | 发球模式(按地址) |
| Ball_Speed | u16 | 球速 km/h |
| Ball_ANGLE1 | u16 | 上下抛球角度 0~90° |
| Ball_ANGLE2 | u16 | 水平抛球角度 0~90°(-45~+45) |
| Ball_FREQ | u16 | 球频 1~10 球/3s |
| Ball_SPIN | u16 | 0~4上旋/5平/6~10下旋 |
| Ball_CYCLE[14] | u16×14 | 循环参数(按地址) |
| Ball_Back | u16 | 反转参数 |
| UP_Speed/DOWN_Speed | u16 | 上/下发球轮百分比 |
| Mag_ANGLE1/Mag_ANGLE2 | u16 | 上下/水平磁铁运动角度 |
`Get_Blue`/`Set_Blue`(motor.C)解析 BLE 帧写入 `BL_Read_DATA` 并置 `*_Refresh`，主循环据此驱动电机。此结构即 App↔固件寄存器映射，与小程序 Modbus 寄存器一一对应。

## 5.5 协议层
- `modbus.h`：`_MOD_FRAMES{addr,function,reg_addr,len,dat[250]}`，`ModBus_unpack`(解析主机请求)、`ModBus_HostPack`(组响应)。Modbus 从站解包器。
- `rs232.h`：`Screen_PRTC{Address,Fun_Code,adds_H/L,Len_H/L}`(读寄存器)、`RS485_PRTC`(485 电机驱动器查询:电压/电流/功率/ERROR/校验和)、`Angle_PRTC`(角度)。`Screen_Unpack/Pack`、`RS485_Pack`、`Screen_Pack_ALL`；`RS485_ADD`/`baudSET`/`Driver_Err`。
- `Mod_APP.C`：`Write_REG`/`Read_REG` 按 `reg_addr&0xff00` 分段(0x0100 设定/0x0200 时间/0x0300 设备ID)，`FLASH_PageWriteE` 页写，`Read_File` 日志读取；寄存器镜像 `reg_in_ram[512]`，标定存 Flash `0x0800F000`。

## 5.6 系统类型（system_dat.h）
位/字节联合体 `TYPE_BIT8/BIT16/BIT_BYTE/BYTES_WORD`；`TYPE_Motor`(12 路推/拉位)、`TYPE_BALL_DATA`(SPEED1/2、RL_ANGLE、UD_ANGLE)、`_SYS_REG`(继电器/数字量/输入寄存器/设备使能/本机地址)。`VERSION 0x020a`。

## 5.7 烧录调试
工程 `CAR/CAR.uvproj`，输出 `CAR/CAR.hex`(181KB)，J-Link 烧录(`CAR/JLinkLog.txt`)。也可经小程序 OTA 升级。
